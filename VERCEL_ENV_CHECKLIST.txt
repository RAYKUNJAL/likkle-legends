   [ ] After adding/updating variables, trigger redeploy

   [ ] Check for any "missing environment variable" warnings

COMMON ISSUES
================================================================================

"PayPal subscription plan not found" during checkout?
→ Check that NEXT_PUBLIC_PAYPAL_PLAN_* variables are set correctly
→ PayPal plan IDs should be in format: P-XXXXXXXXXXXXX

"Emails not sending" after signup?
→ Verify RESEND_API_KEY or BREVO_API_KEY is set
→ Check EMAIL_FROM has valid email format

"AI features not working"?
→ Verify GOOGLE_GENERATIVE_AI_API_KEY or GEMINI_API_KEY is set
→ Check API key is valid and hasn't expired

"Database connection failed"?
→ Verify all three Supabase variables are set
→ Check values are not truncated or malformed

DATABASE TROUBLESHOOTING
================================================================================

If you see "Error: [server-env] Missing required environment variable":

Required Supabase Variables:
- SUPABASE_URL (or NEXT_PUBLIC_SUPABASE_URL)
- SUPABASE_ANON_KEY (or NEXT_PUBLIC_SUPABASE_ANON_KEY)
- SUPABASE_SERVICE_ROLE_KEY (server-side only)

All three must be non-empty. The code will accept either naming convention
(with or without NEXT_PUBLIC_ prefix for URL and anon key).

CUSTOM DOMAIN NOTE
================================================================================

When setting NEXT_PUBLIC_APP_URL for production:
- Development: http://localhost:3000
- Preview/Staging: https://[branch].vercel.app
- Production: https://likkle-legends.com (or custom domain)

This is used for email links and redirects after authentication.
